## i18n híbrid CA + EN

### Arquitectura
- **UI strings** → `src/i18n/ca.json` + `en.json`, accedit via `useT()` hook.
- **Contingut BD** → taula `translations(entity_type, entity_id, lang, value)`.
- **Fallback**: si EN no existeix, retorna CA. Si CA tampoc, fallback al text original.

### Components clau
- `src/i18n/LanguageProvider.tsx`: context React amb `lang`, `setLang`, `t`, `fetchTranslations`, `translateContent`.
- `profiles.language` (text, ca/en): persistit a BD i sincronitzat amb localStorage.
- Funció SQL `get_translation(entity_type, entity_id, lang, fallback)` (per RPC/triggers futurs).

### entity_types suportats
`story_node_title`, `story_node_narrative`, `story_choice_label`, `story_world_name`, `story_world_description`, `story_recipe_name`, `story_recipe_description`, `reward_item_name`.

### Ús al frontend
```tsx
// UI:
const t = useT();
<Button>{t("story.newRun")}</Button>

// Contingut BD (bulk fetch):
const { lang } = useLanguage();
const translations = await fetchTranslations(lang, [
  { entity_type: "story_node_narrative", entity_id: node.id }
]);
const text = translateContent(translations, "story_node_narrative", node.id, node.narrative);
```

### Estat (v1.22.0)
- UI traduïda: ProfilePage, StoryNodeView, WorldMap, ChapterCompleteScreen, InventoryDrawer, PetStatsBar, StoryDeathScreen, StoryEndingScreen, HelpDialog, WhileAwayDialog, DailyChallengeCard, PetEvolutionCard, StoryModePage, RewardReveal.
- Contingut BD: 468 traduccions EN inserides (9 mons, 4 receptes, 100 nodes, 242 choices) via Lovable AI Gateway.
